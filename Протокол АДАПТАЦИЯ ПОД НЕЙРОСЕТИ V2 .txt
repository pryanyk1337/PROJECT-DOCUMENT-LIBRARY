БАЗОВЫЙ СЛОЙ (УНИВЕРСАЛЬНЫЙ):
├─ Работает на ВСЕХ платформах без изменений
├─ Структура: 7 уровней (контекст → роль → интервью → инструкции → ограничения → самопроверка → формат выхода)
├─ Язык: нейтральный (без специфичных конструкций)
└─ Формат: Markdown / обычный текст

СЛОЙ АКТУАЛИЗАЦИИ (ОПЦИОНАЛЬНЫЙ):
├─ Активируется: только по запросу пользователя ("адаптируй под Claude/GPT/Gemini")
├─ Метод: проверка актуальной документации через web_search
├─ Результат: модуль с актуальными данными о платформе
└─ Проверка: валидация информации (дата обновления, источник)

СЛОЙ АДАПТАЦИИ (ОПЦИОНАЛЬНЫЙ):
├─ Частичная адаптация: добавление специфичных элементов
│   ├─ Claude: XML-теги <thinking>, <context>, <answer>
│   ├─ ChatGPT: системные сообщения, Code Interpreter
│   ├─ Gemini: structured outputs, multimodal
│   ├─ Perplexity: цитирование источников [1], [2]
│   └─ Grok: неформальный тон, поиск в X
│
└─ Полная адаптация: переформатирование всей структуры
    ├─ Изменение порядка уровней (если платформа требует)
    ├─ Изменение синтаксиса (JSON → YAML → другой)
    └─ Добавление платформо-специфичных инструкций

ШАГ 1: СБОР ИНФОРМАЦИИ (фаза интервью)
Вопрос: "Для какой нейросети создаём промпт?"
├─ Если "универсальный" → используй БАЗОВЫЙ СЛОЙ
├─ Если "Claude/GPT/Gemini/другая" → переход к ШАГ 2
└─ Если неизвестная платформа → запросить документацию

ШАГ 2: АКТУАЛИЗАЦИЯ ДАННЫХ
IF (платформа указана конкретно):
  1. Проверить актуальность данных через web_search:
     - Текущая версия модели (GPT-5.2, Claude Sonnet 4, Gemini 3)
     - Доступные функции (Spaces, Projects, web_search, Code Interpreter)
     - Особенности синтаксиса (обновления API)
  2. Зафиксировать дату актуализации (ISO 8601)
  3. Создать модуль актуализации (если данные устарели)

ШАГ 3: ВЫБОР ТИПА АДАПТАЦИИ
Вопрос: "Нужна адаптация под платформу?"
├─ Нет → выдать БАЗОВЫЙ промпт
├─ Частичная → добавить модуль специфичных элементов
└─ Полная → переформатировать структуру

ШАГ 4: СОЗДАНИЕ ПРОМПТА
Базовый промпт (7 уровней) + модуль адаптации (если есть)

ШАГ 5: ВАЛИДАЦИЯ
Проверка:
□ Базовая структура сохранена (7 уровней)
□ Адаптация соответствует актуальным данным платформы
□ Промпт работает БЕЗ адаптации (если платформа изменится)

ПРИМЕРЫ МОДУЛЕЙ АДАПТАЦИИ:

## АДАПТАЦИЯ ДЛЯ CLAUDE (Sonnet 4, Opus 4)


Используй XML-теги для структурирования:
<thinking>
[Внутренний анализ задачи, применение метамодели НЛП]
</thinking>

<context>
[Контекст задачи]
</context>

<answer>
[Ответ пользователю]
</answer>

Если используешь Spaces/Projects → контекст сохраняется между сессиями.


## АДАПТАЦИЯ ДЛЯ GEMINI (Gemini 3 Flash/Pro/Thinking)

Используй structured outputs для таблиц и JSON.
Для работы с изображениями применяй multimodal capabilities.
Режим Thinking → используй для глубокого анализа (медленнее, но точнее).
Режим Flash → используй для быстрых задач.


## АДАПТАЦИЯ ДЛЯ CHATGPT (GPT-5.2, 4.1)

Используй системное сообщение для роли и контекста.
Для сложных задач применяй chain-of-thought reasoning.
Если доступен Code Interpreter → используй для вычислений и анализа данных.
Если используешь Projects → сохраняй контекст и файлы.
