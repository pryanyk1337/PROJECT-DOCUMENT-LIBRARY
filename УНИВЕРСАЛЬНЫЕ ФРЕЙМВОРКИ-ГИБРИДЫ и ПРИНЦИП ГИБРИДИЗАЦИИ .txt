ПРАВИЛО 1: ФРЕЙМВОРКИ МОЖНО КОМБИНИРОВАТЬ
├─ Пример: TASK-MASTER (основа) + CREATE (для креативной части)
└─ Результат: структурированный процесс с креативными этапами

ПРАВИЛО 2: ДЛЯ СЛОЖНЫХ ЗАДАЧ = ВСЕ 5 ФРЕЙМВОРКОВ
├─ Этап 1: TASK-MASTER (декомпозиция)
├─ Этап 2: CREATE (генерация вариантов)
├─ Этап 3: ANALYZE (оценка вариантов)
├─ Этап 4: CODE-CRAFT (реализация)
└─ Этап 5: STRATEGY-MAP (интеграция в план)

ПРАВИЛО 3: СОЗДАНИЕ НОВЫХ ГИБРИДОВ
├─ Взять элементы из 2-3 фреймворков
├─ Адаптировать под специфику задачи
└─ Зафиксировать в библиотеке как новый фреймворк


TASK-MASTER (Универсальный для любых задач)

T = Task (Задача): Что нужно сделать?
A = Audience (Аудитория): Для кого результат?
S = Success criteria (Критерии успеха): Как проверить выполнение?
K = Knowledge base (База знаний): Какие данные использовать?
M = Method (Метод): Каким способом выполнить?
A = Autonomy (Автономность): Как решать проблемы самостоятельно?
S = Self-check (Самопроверка): Как проверить качество перед выдачей?
T = Tone (Тон): Формальный/дружелюбный/технический?
E = Example (Пример): Образец результата
R = Restrictions (Ограничения): Что НЕ делать?

CREATE (Креативные задачи)

C = Context (Контекст): Цель и аудитория
R = Role (Роль): Кто агент (дизайнер/писатель/стратег)
E = Explore (Исследование): Собрать референсы и аналогии
A = Alternatives (Альтернативы): Создать минимум 3-5 вариантов
T = Test (Тест): Оценить каждый вариант по критериям
E = Execute (Исполнение): Развить лучший вариант до финала

ANALYZE (Аналитические задачи)

A = Aim (Цель анализа): Что нужно выяснить?
N = Narrow scope (Сузить область): Что конкретно анализируем?
A = Acquire data (Получить данные): Откуда брать информацию?
L = Logic (Логика): Какие методы анализа применить?
Y = Yield insights (Выдать инсайты): Какие выводы сделать?
Z = Zero ambiguity (Устранить неоднозначность): Проверить факты
E = Express (Выразить): Структурировать результаты

CODE-CRAFT (Программирование)

C = Clarify requirements (Уточнить требования): Что должен делать код?
O = Outline structure (Наметить структуру): Архитектура решения
D = Develop core (Разработать ядро): Основная логика
E = Edge cases (Граничные случаи): Обработать ошибки

C = Comment (Комментировать): Документация кода
R = Review (Проверить): Тесты и валидация
A = Adapt (Адаптировать): Под стандарты (PEP8, ESLint)
F = Format (Форматировать): Чистый финальный код
T = Test examples (Примеры тестов): Юнит-тесты

STRATEGY-MAP (Стратегические задачи)

S = Situation (Ситуация): Текущее состояние
T = Target (Цель): Желаемое состояние
R = Resources (Ресурсы): Что доступно (бюджет, время, люди)
A = Actions (Действия): Пошаговый план
T = Timeline (Таймлайн): Сроки этапов
E = Evaluate (Оценить): Метрики успеха
G = Guardrails (Ограничители): Риски и митигация
Y = Yield (Результат): Финальный roadmap
