СТАТУС АГЕНТА "ПРОМПТ ИНЖЕНЕР":
├─ Независимый специализированный агент
├─ НЕ знает инструкций Мета-Агента Архитектора
├─ Имеет собственные проектные инструкции
└─ Может быть встроен в любую систему (Claude Spaces, Projects, GitHub и др.)

ВАРИАНТЫ ИНСТРУКЦИЙ:

ВАРИАНТ А: ПОХОЖИЕ НА МЕТА-АГЕНТА (адаптированные)
├─ Структура: фазы диагностики, создания, проверки
├─ Методология: K/B/T профили (опционально)
└─ Файлы-справочники: собственные (PROMPT_TEMPLATES.md, FRAMEWORK_LIBRARY.md)

ВАРИАНТ Б: УНИКАЛЬНЫЕ (специфичные для промпт-инжиниринга)
├─ Фокус: создание промптов-агентов
├─ Методология: фреймворки промптостроения (RACE, CRISPE, TIER)
└─ Без привязки к системе Мета-Агента

РЕКОМЕНДАЦИЯ: ВАРИАНТ Б (уникальные инструкции)
Почему:
1. Агент решает специфичную задачу (промпт-инжиниринг)
2. Встраивается в любую систему (Claude, GPT, внешние проекты)
3. Не зависит от файлов Мета-Агента
4. Может использоваться независимо

ИНТЕГРАЦИЯ В CLAUDE SPACES:
IF (агент встроен в Claude Space):
  → Использует контекст Space (файлы, история)
  → НО работает автономно (не требует других агентов)
  → Создаёт промпты для любых платформ (не только Claude)
