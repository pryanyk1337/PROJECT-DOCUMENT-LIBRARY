## РАЗДЕЛ 3: ПРАВИЛА ВЕРСИОНИРОВАНИЯ

### 3.1 Схема версий

```
MAJOR.MINOR

MAJOR (X.0): Существенные изменения
- Изменение профиля
- Изменение цели
- Изменение входного/выходного формата

MINOR (X.Y): Незначительные изменения
- Добавление примеров
- Уточнение процесса
- Исправление ошибок
```

### 3.2 Правила обновления

| Изменение | Версия | Пример |
|-----------|--------|--------|
| Новый агент | 1.0 | K-001 v1.0 |
| Добавлен пример | +0.1 | K-001 v1.1 |
| Изменён процесс | +0.1 | K-001 v1.2 |
| Изменён профиль | +1.0 | K-001 v2.0 |
| Изменена цель | +1.0 | K-001 v3.0 |

## РАЗДЕЛ 4: СТАТУСЫ АГЕНТА

| Статус | Критерии | Применение |
|--------|----------|------------|
| **DRAFT** | Не завершена документация | Только разработка |
| **ALPHA** | Документация полная, не тестировался | Внутреннее тестирование |
| **BETA** | Прошёл базовые тесты | Ограниченное использование |
| **RELEASE** | Все тесты пройдены | Полное использование |
| **CERTIFIED** | 10+ успешных применений | Рекомендован |
| **DEPRECATED** | Устарел или заменён | Не использовать |
