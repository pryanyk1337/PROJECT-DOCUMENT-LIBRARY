# INTERVIEW_PHASE_GUIDE.md

## НАЗНАЧЕНИЕ

Полное руководство по проведению фазы интервью для AI-агентов. Включает применение NLP метамодели, типы вопросов, формулировку допущений. Каждый агент ОБЯЗАН провести интервью перед основной работой.

---

## РАЗДЕЛ 1: АРХИТЕКТУРА ФАЗЫ ИНТЕРВЬЮ

```
[ПОЛУЧЕНИЕ ЗАПРОСА]
        ↓
[ФАЗА ИНТЕРВЬЮ]
        ↓
    [Достаточно данных?]
        ↓ ДА              ↓ НЕТ
[ОСНОВНАЯ РАБОТА]    [УТОЧНЯЮЩИЕ ВОПРОСЫ]
                            ↓
                     [ОТВЕТ ПОЛЬЗОВАТЕЛЯ]
                            ↓
                     [ДОСТАТОЧНО?] → ДА → [ОСНОВНАЯ РАБОТА]
                            ↓ НЕТ
                     [ЯВНЫЕ ДОПУЩЕНИЯ] → [ОСНОВНАЯ РАБОТА]
```

## РАЗДЕЛ 2: NLP МЕТАМОДЕЛЬ — ТРИ ФИЛЬТРА

| Фильтр | Что выявляет | Ключевые слова-индикаторы |
|--------|--------------|---------------------------|
| **УПУЩЕНИЯ** | Недостающая информация | "это", "они", "нужно", "сделать" |
| **ОБОБЩЕНИЯ** | Чрезмерно широкие утверждения | "всё", "всегда", "никогда", "должно" |
| **ИСКАЖЕНИЯ** | Непроверенные причинно-следственные связи | "потому что", "из-за", "значит" |

### 2.1 Фильтр упущений

| Паттерн | Пример | Вопрос агента |
|---------|--------|---------------|
| Неопределённое существительное | "Нужна стратегия" | Стратегия чего? (запуска/роста/оптимизации) |
| Неопределённый глагол | "Улучшить продукт" | Улучшить как? (функции/UX/скорость) |
| Отсутствие референта | "Они решили" | Кто именно? |
| Упущенное сравнение | "Нужно лучше" | Лучше чем что? По какому критерию? |
| Отсутствие контекста | "Запустить быстро" | Какой срок? Какие ресурсы? |

### 2.2 Фильтр обобщений

| Паттерн | Пример | Вопрос агента |
|---------|--------|---------------|
| Универсальный квантор | "Проверь ВСЕ факты" | Все = 100% или ключевые? |
| Модальный оператор | "ДОЛЖЕН быть современным" | Что конкретно означает "современный"? |
| Неявное правило | "Сделай правильно" | По каким критериям оценивать? |

### 2.3 Фильтр искажений

| Паттерн | Пример | Вопрос агента |
|---------|--------|---------------|
| Причина-следствие | "Продажи упали из-за дизайна" | Как установлена связь? Есть данные? |
| Эквивалентность | "Низкий трафик = плохой контент" | Проверены другие причины? |
| Пресуппозиция | "Когда запустим рекламу, вырастут продажи" | На чём основано предположение? |

## РАЗДЕЛ 3: ТИПЫ ВОПРОСОВ

### 3.1 Классификация вопросов

| Тип | Назначение | Пример |
|-----|------------|--------|
| **Уточняющий** | Заполнить пробел | "Какой именно продукт?" |
| **Конкретизирующий** | Превратить обобщение в конкретику | "Что значит 'современный'?" |
| **Верифицирующий** | Проверить предположение | "На чём основана связь X→Y?" |
| **Альтернативный** | Предложить варианты | "Вариант А или Б?" |
| **Подтверждающий** | Зафиксировать допущение | "Верно ли, что [допущение]?" |

### 3.2 Формулы вопросов

```
УТОЧНЯЮЩИЙ:
"[Термин] — это [вариант А] или [вариант Б]? Это влияет на [аспект]."

КОНКРЕТИЗИРУЮЩИЙ:
"[Обобщение] — что конкретно имеется в виду? Например: [пример 1], [пример 2]?"

ВЕРИФИЦИРУЮЩИЙ:
"Связь [X] → [Y] требует проверки. Есть ли данные, подтверждающие это?"

ПОДТВЕРЖДАЮЩИЙ:
"Предполагаю: [допущение]. Продолжить с этим параметром?"
```

## РАЗДЕЛ 4: ФОРМУЛИРОВКА ДОПУЩЕНИЙ

### 4.1 Когда формулировать допущения

```
IF пользователь не может уточнить
   OR вопрос остался без ответа
   OR задержка критична
THEN сформулировать явное допущение
```

### 4.2 Структура допущения

```
## [ДОПУЩЕНИЯ]

Следующие параметры не были уточнены. Принимаю:

| Параметр | Допущение | Основание | Влияние на результат |
|----------|-----------|-----------|---------------------|
| [что] | [значение] | [почему выбрано] | [как влияет] |

Продолжить с этими параметрами? (да/нет/уточнить)
```

### 4.3 Пример допущения

```
## [ДОПУЩЕНИЯ]

| Параметр | Допущение | Основание | Влияние |
|----------|-----------|-----------|---------|
| Тип продукта | SaaS | Упомянут "запуск приложения" | Определяет каналы продвижения |
| Срок | 90 дней | Стандарт для MVP | Влияет на объём roadmap |
| Бюджет | $10K | Не указан, принят средний | Ограничивает рекламу |

Продолжить с этими параметрами?
```

## РАЗДЕЛ 5: ЛИМИТЫ ИНТЕРВЬЮ

| Параметр | Лимит | Действие при превышении |
|----------|-------|------------------------|
| Количество вопросов | 2-5 | Сформулировать допущения |
| Итерации уточнений | 2 | Принять лучшее приближение |
| Время на интервью | 10% от задачи | Перейти к работе |

## РАЗДЕЛ 6: ИНТЕГРАЦИЯ В ПРОЦЕСС АГЕНТА

### 6.1 Шаблон фазы интервью для промпта

```
## ФАЗА ИНТЕРВЬЮ (ОБЯЗАТЕЛЬНА)

Шаг 1: Применить фильтры метамодели НЛП:
  - Упущения: есть ли неопределённые термины?
  - Обобщения: есть ли "всё", "всегда", "должно"?
  - Искажения: есть ли непроверенные связи?

Шаг 2: Если найдены неопределённости → задать 2-5 вопросов
  Формат: "[Вопрос] — это влияет на [аспект результата]."

Шаг 3: Если пользователь не уточнил → сформулировать допущения
  Формат: "Предполагаю: [допущение]. Продолжить?"

Шаг 4: СТОП. НЕ переходить к основной работе без подтверждения.
  ЗАПРЕЩЕНО: продолжать без явного "да" или "продолжай".
```

### 6.2 JSON-структура для фазы интервью

```json
"interview_phase": {
  "step_1": "Применить фильтры метамодели (упущения, обобщения, искажения)",
  "step_2": "Задать 2-5 уточняющих вопросов",
  "step_3": "Сформулировать явные допущения при отсутствии ответа",
  "step_4": "СТОП. Ожидать подтверждения. ЗАПРЕЩЕНО продолжать без 'да'",
  "max_questions": 5,
  "max_iterations": 2
}
```

## РАЗДЕЛ 7: ЧЕК-ЛИСТ ФАЗЫ ИНТЕРВЬЮ

```
[ ] Запрос проанализирован через 3 фильтра метамодели
[ ] Выявлены неопределённости (если есть)
[ ] Заданы 2-5 вопросов (или сформулированы допущения)
[ ] Получено подтверждение пользователя
[ ] Только после подтверждения — переход к основной работе
```

## ПРИМЕРЫ ПРИМЕНЕНИЯ

### Пример 1: Полный цикл интервью

```
ЗАПРОС: "Нужна стратегия продвижения."

АГЕНТ (фаза интервью):

## [УТОЧНЕНИЯ]

Применены фильтры метамодели:

1. УПУЩЕНИЕ: "Продвижение" чего? (продукт/услуга/бренд)
2. УПУЩЕНИЕ: Цель продвижения? (узнаваемость/трафик/продажи)
3. УПУЩЕНИЕ: Каналы? (соцсети/SEO/реклама)
4. ОБОБЩЕНИЕ: "Стратегия" — какого масштаба? (1 месяц/квартал/год)

Вопросы:
1. Что продвигаем: продукт, услугу или бренд?
2. Главная цель: узнаваемость, трафик или продажи?
3. Есть предпочтения по каналам или выбрать оптимальные?
4. На какой срок нужна стратегия?

---

ПОЛЬЗОВАТЕЛЬ: "Продукт — мобильное приложение. Цель — скачивания. Срок — 3 месяца."

АГЕНТ: Понял. Переходу к основной работе.
[ОСНОВНАЯ РАБОТА...]
```

### Пример 2: Интервью с допущениями

```
ЗАПРОС: "Напиши статью о криптовалютах."

АГЕНТ (фаза интервью):

## [УТОЧНЕНИЯ]

Выявлены упущения:
1. Тема статьи? (обзор рынка/инвестиции/технология)
2. Объём? (500/2000/5000 слов)
3. ЦА? (новички/инвесторы/технари)

---

ПОЛЬЗОВАТЕЛЬ: "Не знаю, реши сам."

АГЕНТ:

## [ДОПУЩЕНИЯ]

| Параметр | Допущение | Основание | Влияние |
|----------|-----------|-----------|---------|
| Тема | Обзор рынка 2026 | Наиболее универсальная | Широкий охват |
| Объём | 2000 слов | Стандарт для блога | Глубина анализа |
| ЦА | Новички | Максимальная аудитория | Простой язык |

Продолжить с этими параметрами?

---

ПОЛЬЗОВАТЕЛЬ: "Да."

АГЕНТ: [ОСНОВНАЯ РАБОТА...]
```
