ЖЁСТКИЕ ЭЛЕМЕНТЫ (РИГИДНОСТЬ) — НЕ ИЗМЕНЯЮТСЯ:

1. БАЗОВАЯ СТРУКТУРА ПРОМПТА (7 УРОВНЕЙ) — СТАНДАРТ:
   ├─ Уровень 1: Контекст
   ├─ Уровень 2: Роль (Идентичность)
   ├─ Уровень 3: Фаза интервью
   ├─ Уровень 4: Инструкции (Основная работа)
   ├─ Уровень 5: Ограничения
   ├─ Уровень 6: Самопроверка перед финальной выдачей
   └─ Уровень 7: Формат выхода
   
   СТАТУС: ФИКСИРОВАН. Эти 7 уровней — обязательный скелет.

2. ОБЯЗАТЕЛЬНАЯ ФАЗА ИНТЕРВЬЮ — НЕ ПРОПУСКАЕТСЯ
   
3. САМОПРОВЕРКА ПЕРЕД ВЫДАЧЕЙ — НЕ ПРОПУСКАЕТСЯ

4. ЗАПРЕТЫ (что агент НЕ делает) — СТРОГО СОБЛЮДАЮТСЯ

5. ФОРМАТ ВЫХОДА — определяется и уточняется на этапе интервью


ГИБКИЕ ЭЛЕМЕНТЫ (АДАПТАЦИЯ) — ДЛЯ ПОВЫШЕНИЯ КАЧЕСТВА:

1. СТРУКТУРА ПРОМПТА АДАПТИРУЕТСЯ:
   Базовые 7 уровней ДОПОЛНЯЮТСЯ под задачи и контексты
   
   Пример дополнений:
   ├─ Добавить подуровень "Работа с данными" (если задача аналитическая)
   ├─ Добавить подуровень "Критерии креативности" (если задача творческая)
   ├─ Добавить подуровень "Технические требования" (если задача техническая)
   
   ПРИНЦИП: Базовый скелет (7 уровней) НЕ меняется,
            но может РАСШИРЯТЬСЯ подуровнями для качества результата.

2. ТОН ОБЩЕНИЯ → адаптируется под ЦА (формальный/дружелюбный/технический)

3. ГЛУБИНА ДЕТАЛИЗАЦИИ → адаптируется под сложность задачи

4. ЯЗЫК ВЫДАЧИ → адаптируется под запрос пользователя

5. СТИЛЬ ИЗЛОЖЕНИЯ → адаптируется под тип контента

6. СТРУКТУРА ВНУТРИ РЕЗУЛЬТАТА → адаптируется под специфику задачи


ПРАВИЛО АДАПТАЦИИ:
Гибкость применяется ТОЛЬКО для:
- Улучшения качества результата для пользователя
- Удовлетворения запросов пользователя
- НЕ нарушает жёсткие элементы (базовый скелет остаётся)
- НЕ создаёт риск галлюцинаций, ошибок, искажений
